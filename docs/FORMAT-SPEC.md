## Pipeline Overview

```
[1/4] Cagle's scrape  →  data/cagles_inventory_latest.json
[2/4] eBay research   →  data/research_output_latest.json
[3/4] Transform       →  research_data.json  (repo root + docs/)

```

**Entry point:** `python3 scripts/run_pipeline.py`
**Required env:** `EBAY_CLIENT_SECRET`

## 1. Vehicle Selection

### Pool
Source: live Cagle's `inventory.aspx` HTML scrape (header row excluded).

### Selection priority (in order, capped at `max_vehicles=50`)
1. **New arrivals** — vehicle `(year|make|model|row|arrival_date)` not in `data/research_ledger.json`
   - Sorted by `score()` desc (BMW/Mercedes/Cadillac = 12, Lexus/Audi = 9, mainstream = 5, economy = 3-4)
   - `score` adds +3 if model contains: NAV, DVD, SPORT, LUXURY, PREMIUM, TOURING, TURBO, AMG, HYBRID, DENALI, ESCALADE, M3, M5, S4, S5, C63, 335, 435
2. **Backfill** — existing vehicles not researched in **≥7 days**
3. **Top-up** — highest-scored remaining (regardless of age)

### Ledger update
Every researched vehicle appends `vehicle_id(v) → ISO timestamp` to `data/research_ledger.json` on completion.

## 3. Margin Calculation

```
margin = eBay_avg - yard_cost - 12 - (eBay_avg * 0.13 + 0.30)
```

- `12` = flat eBay fees (FVF base + fixed per-order)
- `0.13 * avg + 0.30` = payment processing (PayPal/eBay payments)
- Result clamped at $0 minimum (no negative listed in summaries; shown as `-$X` in cards)

### Per-vehicle stats
- `best_margin` = max margin across all researched parts
- `total_margin` = sum of all positive margins
- `parts_with_data` = parts where eBay returned ≥1 result

### Vehicle badges
- **🆕 NEW** — vehicle not in research_ledger
- **💰 HV** (High-Value) — `best_margin ≥ $500`
- **⏰ LAST** — `days_in_yard ≥ 13` (computed from arrival_date)

    {
      "year": "2014",

      "model": "ESCAPE",
      "yard_row": "17",

      "yard": "Cagle's",
      "best_margin": 1506.73,
      "total_margin": 2298.44,
      "is_new": true,
      "is_hv": false,
      "parts": {
        "<part_name>": {
          "active": {                  // eBay Browse API result
            "avg": 56.90,              // mean of prices
            "median": 46.83,
            "min": 31.00,
            "max": 122.85,
            "count": 20               // number of comparable listings
          },
          "yard_cost": 25,
          "est_margin": 12.20         // computed per §3
        }
      }
    }

## 5. Web App — `docs/index.html`

### Header
- Title: **"Junkyard Hunter's Digest"**
- Subtitle: `Updated: <date>`
- Generated by: "Junkyard Arbitrage Bot"

### Stats bar (4 chips)
| Chip | Source |
|------|--------|
| Total | `stats.total` |
| New | `stats.new` |
| Avg | `stats.avg_margin` (rounded, $-prefixed) |
| Best | `stats.best_margin` (rounded, $-prefixed) |

### Filters (5 buttons, sticky under header)
| Filter | Predicate |
|--------|-----------|
| **All** | `is_new \|\| is_hv \|\| best_margin >= 50` (default visible) |
| **New** | `v.is_new === true` |
| **HV** | `v.is_hv === true` |
| **$50+** | `v.best_margin >= 50` |
| **Watchlist** | hardcoded VIN/make/model list (currently empty; placeholder for future use) |

Plus a **search input** (debounced 300ms, matches make/model substring, case-insensitive).

### Vehicle cards (collapsed view)
- Year/Make/Model header
- Badges row: 🆕 NEW, 💰 HV, ⏰ LAST
- Meta line: `Row <yard_row> · Arrived MM/DD/YYYY · N parts with data`
- Right-side margin: `+$<best_margin>` with "Best" label

### Vehicle cards (expanded — tap to expand)
- All parts as a grid, sorted by `est_margin` desc
- Each part shows: name, signed margin (`+$X` green / `-$X` red / `—` gray), eBay avg (`list $X`)
- Parts with location data show `📍 Tap for details`
- Bottom summary: Total Margin, Parts Researched, Parts With Data

### Part detail modal (tap a part)
- Title: part display name
- Subtitle: `YYYY MAKE MODEL`
- Sections: Research Data (margin + eBay avg), Location, Access, Tools Needed, Time Estimate, Confidence, Removal Steps, Test at Yard, Notes, Factory Upgrade note

### Empty / error states
- **No matches**: hide card grid, show "No vehicles match your filters."
- **Load failure**: show `⚠️ Failed to load research data.<br>Check your connection and try again.`

### Mobile-first design
- Max-width: 100% viewport
- Tap targets ≥ 44px
- Sticky header + filters
- No hover-only affordances (all interactions are tap/click)

## 7. eBay API Behavior

### Authentication
- Endpoint: `https://api.ebay.com/identity/v1/oauth2/token`
- Method: client_credentials
- Scope: `https://api.ebay.com/oauth/api_scope`
- Token TTL: 7200s
- Rate limit: 2s minimum between calls (in-script sleep)

### Browse API call per part
- Endpoint: `https://api.ebay.com/buy/browse/v1/item_summary/search`
- Headers: `Authorization: Bearer <token>`, `X-EBAY-C-MARKETPLACE-ID: EBAY_US`
- Params: `q=<template>`, `limit=20`, `filter=conditionIds:{3000|4000|5000}`
- Template: `{year} {make} {model} <part descriptor>`
- Stats computed: avg, median, min, max, count (from returned prices)

### Failure handling
- HTTP non-200 → part skipped, logged, no entry in research_data
- Empty itemSummaries → part skipped
- All requests wrapped in try/except; never crash the run

### Checkpoints
- Every 5 vehicles, partial results flushed to `data/research_output_latest.json`
- Ledger updated every 5 vehicles
- Ctrl+C → partial results preserved, process exits 0
